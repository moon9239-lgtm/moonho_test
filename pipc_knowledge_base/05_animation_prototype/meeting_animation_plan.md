# PIPC 회의록 애니메이션 보기 기획

## 목표

속기록을 읽기 어려운 사용자가 `애니메이션 보기` 버튼 하나로 회의 흐름을 바로 파악하게 만든다. 위원들이 입장해 착석하고, 속기록 발언 순서대로 말하며, 회의가 끝나면 퇴장하는 형태를 기본 경험으로 둔다.

## 샘플 기준

- 샘플 속기록: `2026-03-25 2026년 제5회 전체회의 속기록`
- 구성: 위원장과 위원 8명, 가운데 후면 사무처 보고자, 우측 안건 목록, 하단 현재 발언 및 전체 발언 흐름
- 현재 구현: 브라우저 기반 즉시 재생 프로토타입
- 참고 영상: 사용자가 준 개인정보보호위원회 전체회의 영상 `https://www.youtube.com/watch?v=jiUVAfrHeUc`

## 핵심 화면

1. 상단 좌측에는 회의 날짜, 회차, 장소, 참석자를 표시한다.
2. 중앙은 회의장 풀샷으로 고정한다.
3. 위원장은 중앙, 위원은 좌우에 배치한다.
4. 사무처 직원은 가운데 앞쪽에서 뒷모습만 보이게 한다.
5. 오른쪽에는 오늘의 안건을 표시하고 현재 진행 안건을 강조한다.
6. 하단에는 현재 발언자와 발언 전문을 표시한다.
7. 하단 오른쪽에는 전체 발언 흐름을 스크롤 목록으로 제공한다.
8. 말하는 캐릭터는 살짝 움직이고 입 모양이 벙긋거리는 효과를 준다.

## 가능 여부

가능하다. 다만 모든 회의를 매번 AI 영상으로 즉시 렌더링하는 방식은 느리고 비용이 크다. 기본은 브라우저 애니메이션으로 즉시 생성하고, 사용자가 내보내기를 원할 때만 영상 렌더링을 돌리는 방식이 가장 좋다.

## 추천 구조

1. 속기록 파서
   - PDF/마크다운 속기록에서 회의 메타데이터, 안건, 발언자, 발언문을 추출한다.
   - 발언 원문은 바꾸지 않고 줄바꿈과 페이지 번호만 정리한다.

2. 발언자 매핑
   - `위원장 송경희` 같은 표기를 캐릭터 ID와 좌석 위치에 연결한다.
   - 위원이 아닌 `기획조정관`, `조사조정국장`, `심사총괄담당관` 등은 사무처 보고자 캐릭터로 묶는다.

3. 장면 JSON
   - 회의마다 `meeting_data.json`을 만든다.
   - 포함 항목: 회의 제목, 날짜, 장소, 참석자, 안건, 발언 배열, 캐릭터 에셋 경로, 좌석 위치.

4. 브라우저 플레이어
   - JSON을 읽어서 회의장, 안건, 대사, 타임라인을 렌더링한다.
   - 재생, 일시정지, 이전/다음 발언, 속도 조절, 타임라인 클릭 이동을 제공한다.

5. 캐시
   - 처음 버튼을 누르면 파싱하고 JSON을 저장한다.
   - 다음부터는 저장된 JSON으로 즉시 연다.

6. 선택적 영상화
   - 공유용 MP4가 필요할 때만 브라우저 화면 녹화 또는 AI 비디오 렌더링을 실행한다.
   - 회의 전체를 AI 영상으로 만들기보다 안건 단위나 핵심 구간 단위로 렌더링하는 것이 현실적이다.

## MVP 버튼 흐름

1. 회의 목록 카드에 `애니메이션 보기` 버튼을 추가한다.
2. 버튼 클릭 시 해당 회의의 transcript path를 찾는다.
3. 이미 생성된 JSON이 있으면 바로 viewer를 연다.
4. 없으면 파서를 실행해 JSON을 만든 뒤 viewer를 연다.
5. viewer는 브라우저 애니메이션을 즉시 재생한다.
