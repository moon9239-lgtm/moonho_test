# 보호위 전체회의 내부 업무도구 설계

## 목표

개인정보보호위원회 내부 실무자가 전체회의 운영 현황을 한눈에 파악하고, 특정 회의의 속기록과 안건 내용을 깊게 확인하며, 위원별 성향 분석과 새 안건 준비까지 이어갈 수 있는 내부 업무도구를 만든다.

첫 버전은 대국민 공개 서비스가 아니다. 공개형 회의 탐색 기능은 이후 일부 화면을 분리해 확장할 수 있지만, 1차 제품의 기준은 내부 실무자의 회의 현황 보고와 회의 준비 업무다.

## 1차 사용자

- 전체회의 현황을 파악하거나 보고해야 하는 보호위 내부 실무자
- 특정 회의의 속기록, 안건, 의결서, 관련 법조항을 확인해야 하는 실무자
- 새 안건 상정 전 유사 안건, 예상 쟁점, 위원별 예상 질문을 준비해야 하는 실무자

향후 보조 사용자로 일반 국민과 개인정보 실무자를 고려할 수 있다. 다만 1차 설계에서는 내부용 기능과 공개 가능 기능을 분리할 수 있는 구조만 확보한다.

## 제품 구조

앱은 네 개의 주요 작업 영역으로 나눈다.

1. 회의 운영 상황판
2. 회의 상세 탐색
3. 위원별 분석
4. 새 안건 준비 도우미

기본 진입 화면은 회의 운영 상황판이다. 나머지 작업 영역은 네비게이션과 상황판의 회의 카드, 상세 화면의 근거 링크를 통해 이동할 수 있어야 한다.

## 회의 운영 상황판

회의 운영 상황판은 보호위 출범 이후 전체 기간을 기본 범위로 보여준다. 처음 화면은 특정 연도가 아니라 전체 흐름을 보여주고, 사용자가 필요하면 연도나 기간으로 좁힐 수 있어야 한다.

화면 톤은 보고서형 설명 페이지가 아니라 내부 업무용 상황판이다. 큰 숫자, 상태 신호, 짧은 설명, 바로 눌러 들어갈 수 있는 회의 카드가 중심이 된다.

핵심 지표는 회의 운영 중심으로 구성하되, 세부 분해는 안건 처리 기준을 함께 보여준다.

- 총 회의 수
- 총 안건 수
- 회의당 평균 안건 수
- 연도별 회의 수
- 의결, 보고, 심의, 기타 등 안건 처리 유형 분포
- 공개/비공개 비율
- 최신 데이터 업데이트 기준일

자료 완성도는 주요 KPI로 노출하지 않는다. 회의록, 속기록, 의결서, 안건 메타데이터는 내부적으로 완비되어 있다는 전제로 둔다. 화면에는 필요한 경우 `YYYY년 MM월 DD일 기준 업데이트`처럼 낮은 강조도로 표시한다.

상황판 하단에는 압축형 회의 카드 목록을 둔다. 각 카드는 빠른 스캔이 목적이므로 다음 정보만 간결하게 보여준다.

- 회차 또는 회의명
- 개최일
- 안건 수
- 주요 처리 유형
- 대표 키워드
- 제재·처분 신호 배지

특이 신호는 제재·처분을 우선한다. 과징금, 과태료, 시정명령, 공표명령, 대형 유출, 사회적 파급이 큰 처분, 처분 수위나 금액이 큰 안건이 포함된 회의는 카드에서 바로 눈에 띄어야 한다.

## 회의 상세 탐색

회의 카드를 클릭하면 회의 상세 탐색 화면으로 이동한다. 기본 화면은 애니메이션이 아니라 속기록과 안건을 정확하게 확인하는 업무형 화면이다.

회의 상세 화면은 다음 정보를 포함한다.

- 회의 메타데이터
- 안건 목록
- 속기록 원문
- 발언자와 안건 구간 정보
- 회의록, 의결서, 원천 문서 링크
- 선택된 발언의 관련 법조항 패널

속기록 안의 법조항은 클릭 가능해야 한다. 법조항을 클릭하면 회의일 기준 조문과 현재 조문을 나란히 비교한다. 이 기능은 `korean-law-mcp`를 활용한다. 구현 단계에서는 법령명, 조문 번호, 회의일을 기준으로 과거 조문을 조회할 수 있는지 확인해야 한다.

## 선택형 회의 애니메이션 모드

애니메이션은 회의 상세 화면의 기본값이 아니다. 회의 상세 화면 오른쪽 상단에 `애니메이션으로 보기` 버튼을 두고, 사용자가 클릭했을 때만 해당 회의를 회의 재현 모드로 전환한다.

애니메이션은 짧은 요약 영상이 아니라 하나의 회의를 기준으로 속기록 전체를 재현하는 모드다. 목표 흐름은 다음과 같다.

- 위원 입장 또는 착석
- 개회
- 안건 상정
- 질의응답
- 보고 또는 의결
- 산회
- 퇴장

속기록 텍스트는 애니메이션과 함께 확인할 수 있어야 한다. 사용자가 속기록 발언을 클릭하면 애니메이션이 해당 발언 시점으로 바로 이동한다.

애니메이션 모드는 다음 데이터를 사용한다.

- 회의 메타데이터: 날짜, 회차, 안건, 참석자
- 발언 단위 속기록: 발언자, 순서, 발언 내용, 안건 구간
- 이벤트 데이터: 개회, 안건 전환, 질의응답, 보고, 의결, 산회
- 법조항 참조: 발언 내 조항, 당시 조문, 현재 조문
- 위원 캐릭터 자산: 위원 ID, 이름, 역할, 임기, 사진 참조, 캐릭터 이미지 또는 스프라이트, 좌석 정보
- 장면 매핑: 발언 ID와 애니메이션 타임라인 위치

기존 파일 중 `pipc_knowledge_base/04_members/character_profiles`와 그 하위 `photo_references`에는 위원 사진 참조와 캐릭터 변환 파일이 있을 가능성이 있다. 구현 전 파일 확인 단계에서 이 폴더를 우선 점검한다.

캐릭터 자산이 없는 위원은 내부 QA용 기본 아바타를 사용할 수 있다. 다만 이 상태는 사용자에게 완성된 캐릭터처럼 보이면 안 되고, 구현 검증 단계에서 누락 자산으로 추적되어야 한다.

## 위원별 분석

위원별 분석은 별도 내부 메뉴로 제공한다. 목적은 실무자가 과거 속기록을 바탕으로 위원별 질문 패턴과 관심 관점을 파악해 회의 준비에 활용하는 것이다.

위원별 분석은 정량 근거, 정성 해석, 회의 준비용 예상 질문을 함께 보여준다.

정량 정보:

- 발언량
- 질문 빈도
- 자주 언급한 쟁점
- 자주 언급한 법조항
- 안건 유형별 발언 분포
- 질문 또는 검토 요청 유형

정성 정보:

- 보호 관점이 강한지
- 산업진흥 또는 현장 부담을 함께 고려하는지
- 절차와 증거를 중시하는지
- 기술적 안전조치와 보안 조치를 중시하는지
- 공공기관, 민간기업, 플랫폼, 유출, AI 등 특정 영역에 관심이 집중되는지

정성 해석은 반드시 속기록 근거와 함께 보여준다. 단정적인 라벨만 제공하지 않고, 관련 발언과 회의로 돌아갈 수 있는 링크를 제공해야 한다.

위원별 분석 결과는 새 안건 준비 도우미에서 위원별 예상 질문을 생성하는 근거로도 사용한다.

## 새 안건 준비 도우미

새 안건 준비 도우미는 내부 실무자용 기능이다. 1차 입력 방식은 파일 업로드가 아니라 텍스트 입력이다.

실무자는 안건명과 요약 내용을 입력한다. 시스템은 기존 회의와 안건 자료를 바탕으로 다음 결과를 제공한다.

- 유사 과거 안건
- 유사 법조항
- 과거 처분 수준
- 예상 쟁점
- 회의 전 확인해야 할 포인트
- 위원별 예상 질문
- 준비 체크리스트

필요하면 결과 화면에서 가상 회의 시뮬레이션을 실행할 수 있다. 이 시뮬레이션은 실제 과거 회의 재현과 다르다. 새 안건 요약, 유사 과거 안건, 위원별 분석을 바탕으로 가능한 질의응답 흐름을 구성하는 내부 준비용 기능이다.

파일 업로드, 검토보고서 자동 파싱, 의결서 초안 분석은 이후 확장으로 둔다. 첫 버전은 텍스트 입력만으로 실무 효용을 검증한다.

## 데이터 원천과 색인

기존 프로젝트 파일을 최대한 재활용한다. 우선 확인해야 할 경로는 다음과 같다.

- `pipc_knowledge_base/99_raw/transcripts`
- `pipc_knowledge_base/01_meetings`
- `pipc_knowledge_base/02_decisions`
- `pipc_knowledge_base/04_members`
- `pipc_knowledge_base/04_members/character_profiles`
- `pipc_knowledge_base/04_members/character_profiles/photo_references`
- `pipc_dashboard/data/dashboard-data.js`
- `supabase/migrations`

데이터는 다음 단위로 정규화되어야 한다.

- 회의
- 안건
- 발언
- 법조항 참조
- 위원
- 의결서 또는 처분
- 애니메이션 장면

회의 운영 상황판은 회의·안건 집계 색인을 사용한다. 회의 상세는 발언 단위 속기록 색인을 사용한다. 위원별 분석은 발언 단위 색인을 위원 기준으로 다시 집계한다. 새 안건 준비 도우미는 사용자가 입력한 안건명과 요약을 과거 안건, 의결서, 법조항, 위원 발언 색인과 비교한다.

## MVP 범위

1차 MVP에 포함한다.

- 전체 기간 기본값을 가진 회의 운영 상황판
- 회의 운영과 안건 처리 유형을 함께 보여주는 KPI와 차트
- 제재·처분 신호 배지가 있는 압축형 회의 카드
- 속기록, 안건, 문서 링크, 법조항 비교 구조를 가진 회의 상세 화면
- 대표 법조항에 대한 `korean-law-mcp` 당시 조문·현재 조문 비교 검증
- 선택된 회의 1개를 기준으로 한 속기록 기반 애니메이션 재현 프로토타입
- 정량 근거, 정성 해석, 근거 발언 링크를 포함한 위원별 분석 초기 버전
- 텍스트 입력 기반 새 안건 준비 도우미
- 새 안건에 대한 유사 안건, 유사 조항, 처분 수준, 예상 쟁점, 위원별 예상 질문, 준비 체크리스트

1차 MVP에서 미룬다.

- 파일 업로드 기반 안건 분석
- 모든 회의의 완전 자동 애니메이션화
- 법령 개정 이력 전체 탐색
- 운영 환경 수준의 권한관리와 인증
- 외부 공개용 화면

미루는 기능도 향후 확장을 막지 않도록 메뉴, 데이터, 화면 경계를 분리한다.

## 검증 기준

- 실무자가 첫 화면에서 전체회의 운영 현황을 1분 안에 파악할 수 있다.
- 회의 카드에서 특정 회의 상세로 이동해 속기록과 안건 정보를 잃지 않고 확인할 수 있다.
- 대표 법조항에 대해 회의 당시 조문과 현재 조문을 비교할 수 있다.
- 선택된 회의 1개에 대해 개회부터 산회까지 속기록 기반 애니메이션 재현 방향을 확인할 수 있다.
- 위원별 분석은 해석과 함께 근거 발언을 제공한다.
- 새 안건 요약 입력만으로 회의 준비에 쓸 수 있는 유사 안건, 관련 조항, 예상 쟁점, 위원별 예상 질문이 나온다.

## 구현 전 확인 항목

- 위원별 캐릭터 이미지 또는 스프라이트가 실제로 어떤 경로에 존재하는지 확인한다.
- `korean-law-mcp`가 회의일 기준 과거 조문 조회와 현재 조문 조회를 모두 지원하는지 확인한다.
- 기존 Supabase 뷰가 회의, 안건, 속기록, 의결서, 위원 정보를 새 설계에 맞게 제공하는지 확인한다.
- 현재 `pipc_dashboard` 파일을 계속 고도화할지, 기존 데이터를 재료로 삼아 UI를 재구성할지 결정한다.
